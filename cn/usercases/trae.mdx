## 概述

TRAE 是一个面向开发流程的 AI 工具或平台，通常用于问答、生成、分析和协作式编码。

将 TRAE 接入 HubTo 后，你可以统一管理 API Key、模型权限和调用额度，并在同一入口下切换不同模型。

## 适用场景

- 你希望把 TRAE 的模型访问统一到团队网关
- 你希望在不同成员之间分配不同的访问权限
- 你希望在不改核心工作流的前提下替换模型

## 前置条件

- 已安装或可访问 TRAE
- 已拥有可用的 HubTo API Key
- 网络环境可以访问 `https://api.hubto.ai`

## 配置步骤

### 步骤 1：查找模型或 Provider 设置

在 TRAE 的后台、设置页或工作区配置中，找到模型供应方、自定义 API 或 Base URL 配置入口。

### 步骤 2：使用兼容方式接入

如果 TRAE 支持 OpenAI 兼容接口或自定义模型供应方，可以直接接入 HubTo。

### 步骤 3：填写推荐参数

- Name：`HubTo`
- Base URL：`https://api.hubto.ai/v1`
- API Key：`YOUR_API_KEY`
- Model：例如 `deepseek-v3.1`、`glm-5.1`

### 步骤 4：进行测试

配置完成后，执行一个简单任务，例如“总结这个仓库的目录结构”，确认调用结果正常。

## 使用建议

- 优先验证最基础的对话或问答能力
- 若 TRAE 支持多个工作区，建议按工作区拆分 API Key
- 如果团队里有不同的模型需求，建议在 HubTo 中先完成模型权限划分

## 常见问题

### 找不到自定义模型入口

有些版本可能默认只展示官方供应方入口，这时可以检查高级设置或工作区级配置。

### 返回 401 或 403

请检查 API Key 是否有效，以及当前令牌是否已开通对应模型权限。

### 请求路径错误

请确认 Base URL 使用 `https://api.hubto.ai/v1`。
