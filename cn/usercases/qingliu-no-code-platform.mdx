## 概述

轻流无代码开发平台适合把表单、流程和业务规则快速组合成可执行应用。

将 HubTo 接入后，你可以在审批、表单处理、知识问答或文本生成等场景中复用统一模型网关。

## 适用场景

- 你希望在无代码流程中加入文本生成、总结或分类能力
- 你希望业务团队直接复用统一的模型接入方式
- 你希望通过平台变量管理 API Key 与请求地址

## 前置条件

- 已部署或可访问轻流无代码开发平台
- 已拥有可用的 HubTo API Key
- 当前平台支持 HTTP 请求、Webhook 或扩展调用

## 配置步骤

### 步骤 1：确定接入入口

优先查找平台中的 HTTP 请求、Webhook、扩展插件或 AI 配置节点。

### 步骤 2：写入 HubTo 请求参数

推荐使用以下信息：

- Base URL：`https://api.hubto.ai/v1/chat/completions`
- Authorization：`Bearer YOUR_API_KEY`
- Model：例如 `deepseek-v3.1`、`glm-5.1`

### 步骤 3：构造请求体

可以先从一个最简单的文本生成请求开始：

```json
{
  "model": "deepseek-v3.1",
  "messages": [
    {
      "role": "user",
      "content": "请根据表单内容生成一段客户回访摘要"
    }
  ]
}
```

### 步骤 4：保存并测试

在测试流程中提交一条样本数据，确认平台可以成功拿到模型返回内容。

## 使用建议

- 建议把 API Key 存在平台密钥或环境变量里，不要直接写死在表单配置中
- 如果你有多个业务流程，建议统一维护一个 HubTo 请求模板
- 如果输出内容需要进一步流转，建议在平台里先做长度、空值和异常兜底处理

## 常见问题

### 请求被平台拦截

请确认平台的 HTTP 节点允许自定义 Header，并支持向外部地址发起请求。

### 返回认证错误

请确认 Authorization 头格式为 `Bearer YOUR_API_KEY`。

### 响应内容解析不到结果

请检查你读取的字段路径是否与接口响应结构一致。
