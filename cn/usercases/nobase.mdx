## 概述

NoBase 适合把应用逻辑、数据处理和自动化能力组合成轻量工作流。

将 HubTo 接入后，你可以把文本生成、分类、总结和问答能力统一接入到 NoBase 的应用流程中。

## 适用场景

- 你希望在 NoBase 中快速接入统一模型网关
- 你希望在业务流程中复用文本生成或结构化处理能力
- 你希望通过平台变量管理 API Key 和模型参数

## 前置条件

- 已部署或可访问 NoBase
- 已拥有可用的 HubTo API Key
- 当前版本支持 HTTP 请求、扩展能力或自定义模型服务

## 接入方式

### 方式一：直接配置模型服务

如果 NoBase 支持配置模型供应方或兼容 OpenAI 的接口，可以直接填写：

- Base URL：`https://api.hubto.ai/v1`
- API Key：`YOUR_API_KEY`
- Model：例如 `deepseek-v3.1`、`glm-5.1`

### 方式二：通过 HTTP 请求调用

如果你当前是通过流程节点接入 AI 能力，也可以在节点中直接调用 `https://api.hubto.ai/v1/chat/completions`。

## 推荐验证方式

建议先做一个最小实验，例如输入一段文本并返回一条摘要，确认结果可以正确流入后续节点。

## 使用建议

- 建议把 API Key 放在平台密钥管理中，而不是硬编码在节点里
- 如果你有多个应用，可以沉淀一个公共 HubTo 调用模板
- 当输出会驱动后续逻辑时，建议加入失败重试和空值兜底

## 常见问题

### 响应回来了，但流程没有继续

请检查节点映射是否读取了正确的结果字段。

### 请求一直超时

请确认 NoBase 所在环境可以访问 `https://api.hubto.ai/v1`。

### 模型名称无效

请确认当前使用的模型名与 HubTo 中已开通的模型一致。
