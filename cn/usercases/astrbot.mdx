## 概述

AstrBot 适合搭建机器人、频道助手和自动化对话能力。

将 HubTo 接入后，你可以为不同机器人统一配置模型访问入口，并集中管理 API Key、权限和配额。

## 适用场景

- 你希望在 AstrBot 中统一管理模型调用
- 你希望按机器人或频道隔离 API Key
- 你希望后续切换模型时尽量不改动机器人逻辑

## 前置条件

- 已部署或可访问 AstrBot
- 已拥有可用的 HubTo API Key
- 当前版本支持模型供应方、自定义 API 或 HTTP 请求能力

## 配置步骤

### 步骤 1：进入机器人或模型配置

找到 AstrBot 中的模型设置、Provider 配置或工作流节点入口。

### 步骤 2：填写 HubTo 参数

- Base URL：`https://api.hubto.ai/v1`
- API Key：`YOUR_API_KEY`
- Model：例如 `deepseek-v3.1`、`glm-5.1`

### 步骤 3：发起测试消息

例如让机器人回答“请介绍一下 HubTo 可以解决什么问题”，确认返回正常。

## 使用建议

- 不同机器人建议使用不同 API Key，便于统计和审计
- 先验证单轮对话，再逐步接入多轮记忆或复杂流程
- 如果机器人需要固定风格，建议把角色设定放在提示词层而不是写死在代码里

## 常见问题

### 机器人不回复

请先确认消息接入链路正常，再检查模型调用日志。

### 返回 401

请确认 API Key 已注入到 AstrBot 的运行环境，并且值没有多余空格。

### 输出质量不稳定

可以优先切换到更适合对话或指令跟随的模型，并调整机器人提示词。
