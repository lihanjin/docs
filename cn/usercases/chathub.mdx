## 概述

ChatHub 是一个聚合多模型的聊天工具。

接入 AllTokens 后，你可以在 ChatHub 中统一使用 AllTokens 提供的模型与网关能力，并复用同一套 API Key 管理方式。

## 适用场景

- 你希望在一个客户端中集中使用多个模型
- 你希望把 ChatHub 的模型调用统一收敛到 AllTokens
- 你希望在不改使用习惯的情况下切换模型或控制额度

## 前置条件

- 已安装 ChatHub
- 已准备好 AllTokens API Key
- 可以访问 `https://alltoken.aatest.online`

## 配置步骤

### 步骤 1：打开模型提供方配置

在 ChatHub 中进入模型或提供方设置页面，选择支持自定义接口的提供方。

如果你看到 `OpenAI Compatible`、`OpenAI API` 或 `自定义 OpenAI` 一类选项，优先选择它。

### 步骤 2：填写连接信息

填写以下信息：

- API Key：`YOUR_API_KEY`
- Base URL：`https://alltoken.aatest.online/v1`
- Model：例如 `glm-4`、`deepseek-v3`、`claude-3-5-sonnet` 或其他你已开通的模型

### 步骤 3：保存并测试

保存配置后，新建一段对话并发送一条简单消息，确认可以正常返回内容。

## 推荐验证方式

建议先使用一个短问题进行测试，例如“请用一句话介绍 AllTokens”。如果能够正常返回，再切换到你的目标模型或正式对话。

## 使用建议

- 优先选择 OpenAI 兼容方式接入
- 先用一个基础模型验证链路，再切换到目标模型
- 如需多人共用，建议为 ChatHub 单独创建一个 API Key

## 常见问题

### 提示认证失败

请确认 API Key 使用的是 AllTokens 控制台中生成的有效令牌。

### 提示模型不存在

请检查模型名称是否填写正确，并确认当前令牌具备该模型访问权限。

### 保存后仍然调用旧配置

部分版本的 ChatHub 会缓存提供方配置。修改后如果没有生效，建议重新打开会话，或者重启 ChatHub 后再试一次。
