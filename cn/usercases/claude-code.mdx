## 概述

Claude Code 是一个面向开发者的命令行 AI 助手，适合代码生成、调试、重构和代码库问答。

将 Claude Code 接入 Hubto 后，你可以把请求统一走同一套网关地址，并集中管理 API Key、模型权限、额度和路由策略。

## 适用场景

- 你希望把 Claude Code 的调用统一纳入团队的模型网关
- 你希望在不改日常使用习惯的情况下切换模型或管理额度
- 你希望为开发环境单独创建 API Key，便于审计和隔离

## 前置条件

- 已安装并可以正常使用 Claude Code
- 已拥有可用的 Hubto API Key
- 网络环境可以访问 `https://hubto.ai`

## 方法一：通过环境变量配置

### 步骤 1：配置 API 认证

在终端中设置你的 API Key：

```bash
export ANTHROPIC_API_KEY="YOUR_API_KEY"
```

### 步骤 2：配置网关地址

将 Claude Code 的基础地址指向 Hubto：

```bash
export ANTHROPIC_BASE_URL="https://hubto.ai"
```

### 步骤 3：选择模型

如果你希望默认使用某个 Claude 模型，可以继续设置。常见示例：

```bash
export ANTHROPIC_MODEL="claude-3-5-sonnet"
```

### 步骤 4：启动 Claude Code

完成环境变量配置后，正常启动 Claude Code 即可：

```bash
claude
```

## 方法二：写入 shell 配置

如果你希望每次打开终端都自动生效，可以把这些配置写入 `~/.zshrc` 或 `~/.bashrc`：

```bash
export ANTHROPIC_API_KEY="YOUR_API_KEY"
export ANTHROPIC_BASE_URL="https://hubto.ai"
export ANTHROPIC_MODEL="claude-3-5-sonnet"
```

保存后重新加载 shell：

```bash
source ~/.zshrc
```

## 验证配置

完成配置后，可以直接在 Claude Code 中发起一个简单请求，例如让它解释当前目录中的一个文件，确认能够正常返回结果。

如果你切换了模型，也建议先发起一条短请求，确认当前令牌具备目标模型的访问权限。

## 使用建议

- 先在 Hubto 控制台中确认令牌已启用 Claude 相关模型
- 如需切换模型，只需要修改 `ANTHROPIC_MODEL`
- 如果你同时管理多个环境，建议为 Claude Code 单独创建一个 API Key
- 如果你和团队共用网关，建议区分开发、测试和生产环境的令牌

## 常见问题

### 提示认证失败

请确认 `ANTHROPIC_API_KEY` 使用的是 Hubto 控制台里生成的有效令牌，并且值本身没有多余空格或引号问题。

### 提示找不到模型

请确认当前令牌具备该模型的访问权限，并且模型名称填写正确。

### 请求没有走 Hubto

请确认 `ANTHROPIC_BASE_URL` 已设置为：

```txt
https://hubto.ai
```

### 修改配置后没有生效

请确认已经重新加载 shell 配置，或者重新打开了终端会话。
