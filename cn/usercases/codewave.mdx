## 概述

CodeWave 适合把 AI 能力融入应用生成、逻辑编排或开发辅助流程中。

将 HubTo 接入后，你可以使用统一的 API Key、模型名和基础地址，减少多供应方配置带来的维护成本。

## 适用场景

- 你希望在 CodeWave 中集中管理模型调用
- 你希望通过同一套网关切换不同上游模型
- 你希望在项目级别隔离 Key 与访问权限

## 前置条件

- 已部署或可访问 CodeWave
- 已拥有可用的 HubTo API Key
- 当前版本支持自定义模型接口或 HTTP 请求能力

## 接入方式

### 方式一：通过模型配置接入

如果 CodeWave 提供了模型供应方、API 配置或自定义 Provider 入口，可以直接填写：

- Provider：`HubTo`
- Base URL：`https://api.hubto.ai/v1`
- API Key：`YOUR_API_KEY`
- Model：例如 `deepseek-v3.1`、`glm-5.1`

### 方式二：通过脚本或 HTTP 节点调用

如果你是通过流程节点接入 AI 能力，也可以直接调用 HubTo 的 OpenAI 兼容接口。

## 推荐验证方式

建议先做一个最小测试，例如生成一段说明文本或总结一条输入内容，确认链路打通后再接入复杂流程。

## 使用建议

- 先用一个通用模型验证，再扩展到更复杂的推理或多模态模型
- API Key 建议保存在平台密钥或环境变量中
- 若 CodeWave 支持复用模板，建议把 HubTo 配置沉淀为公共模板

## 常见问题

### 平台里没有自定义模型入口

可以先检查是否支持脚本节点、Webhook 或 HTTP 请求能力，作为替代接入方式。

### 请求报 404

请确认接口地址填写为 `https://api.hubto.ai/v1` 或具体端点地址，而不是站点首页。

### 模型切换后无返回

请确认目标模型已开通，并检查原有缓存配置是否仍然引用旧模型。
