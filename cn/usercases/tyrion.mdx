## 概述

Tyrion 是一个面向开发与自动化协作场景的 AI 工具，可用于代码问答、辅助生成和任务执行。

接入 HubTo 后，你可以统一配置模型访问地址、API Key 和权限策略，减少多套上游配置带来的维护成本。

## 适用场景

- 你希望在 Tyrion 中使用团队统一的模型网关
- 你希望根据项目或成员划分不同的调用令牌
- 你希望在后续切换模型时尽量不改动本地工作流

## 前置条件

- 已安装或可访问 Tyrion
- 已拥有可用的 HubTo API Key
- 网络环境可以访问 `https://api.hubto.ai`

## 配置步骤

### 步骤 1：打开模型接入设置

在 Tyrion 中找到 Provider、模型、API 或高级配置区域。

### 步骤 2：选择 OpenAI 兼容或自定义接口

如果支持自定义 Base URL、API Key 和模型名称，就可以直接接入 HubTo。

### 步骤 3：填写推荐配置

- Provider：`HubTo`
- Base URL：`https://api.hubto.ai/v1`
- API Key：`YOUR_API_KEY`
- Model：例如 `deepseek-v3`、`glm-4`

### 步骤 4：验证调用

保存后执行一个简短任务，确认 Tyrion 已经能通过 HubTo 正常返回结果。

## 使用建议

- 建议先接入一个通用模型，确认链路没问题后再扩展更多模型
- 如果 Tyrion 支持项目级配置，建议为不同项目分别维护 API Key
- 团队共享时，建议在 HubTo 中提前设置好权限与配额

## 常见问题

### 工具里没有展示自定义配置

可以先检查是否需要开启实验功能，或者切换到高级设置模式。

### 返回模型不存在

请确认模型名称填写正确，并确认当前令牌已拥有相应权限。

### 配置完成后仍然请求旧地址

请确认原有 Provider 已被禁用，或重新启动工具使配置生效。
