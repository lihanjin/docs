## 概述

LazyCraft 适合搭建工作流、任务编排和智能体类应用。

将 HubTo 接入后，你可以把模型调用统一收敛到同一套网关地址，并集中管理 API Key、权限和额度策略。

## 适用场景

- 你希望在 LazyCraft 中复用统一模型网关
- 你希望把不同工作流的模型调用配置沉淀为公共模板
- 你希望在切换模型时减少对现有流程的改动

## 前置条件

- 已部署或可访问 LazyCraft
- 已拥有可用的 HubTo API Key
- 当前版本支持自定义模型服务、HTTP 节点或兼容接口接入

## 接入步骤

### 步骤 1：确认工作流中的 AI 接入方式

优先查看 LazyCraft 是否支持模型供应方配置。如果没有，也可以查找 HTTP 请求节点或插件扩展入口。

### 步骤 2：填写 HubTo 推荐配置

- Base URL：`https://api.hubto.ai/v1`
- API Key：`YOUR_API_KEY`
- Model：例如 `deepseek-v3`、`glm-4`

### 步骤 3：创建一个最小工作流验证

例如输入一段文本并输出摘要，确认 LazyCraft 可以成功调用 HubTo 返回结果。

## 使用建议

- 建议先搭建一个最小流程模板，再复制到正式业务里
- 如果你有多条工作流，建议把 API Key 和模型名做成公共变量
- 若需要稳定切换不同模型，优先在 HubTo 中调整权限与路由

## 常见问题

### 工作流节点请求失败

请确认当前节点支持自定义 Header，并检查 `Authorization` 头是否正确注入。

### 返回内容不能进入下一个节点

请检查节点映射和字段读取路径是否与接口响应结构一致。

### 模型响应慢

建议先用更轻量的模型验证流程，再根据业务场景切换到目标模型。
