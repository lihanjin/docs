## 概述

LangBot 是一个面向机器人、对话和自动化消息处理场景的框架或平台。

将 LangBot 接入 HubTo 后，你可以统一配置模型访问地址、API Key 与权限策略，便于在不同 Bot 场景中复用同一套模型接入方式。

## 适用场景

- 你希望在 Bot 或消息处理链路中统一调用模型
- 你希望按机器人、频道或工作区拆分 API Key
- 你希望在后续切换模型时不改动大量业务逻辑

## 前置条件

- 已部署或可访问 LangBot
- 已拥有可用的 HubTo API Key
- 当前版本支持自定义模型供应方、HTTP 接口或插件扩展

## 配置步骤

### 步骤 1：确认接入入口

优先查找 Provider、模型配置、OpenAI 兼容入口，或工作流中的 HTTP 请求节点。

### 步骤 2：填写 HubTo 参数

- Base URL：`https://api.hubto.ai/v1`
- API Key：`YOUR_API_KEY`
- Model：例如 `deepseek-v3.1`、`glm-5.1`

### 步骤 3：配置一个测试机器人

例如让机器人回复“请用一句话介绍 HubTo”，确认消息链路可以正常返回模型内容。

## 使用建议

- 建议为不同机器人使用不同 API Key，便于追踪用量
- 若 LangBot 支持系统提示词，建议把通用角色设定放在 Bot 层，模型能力留给 HubTo 管理
- 若有高并发场景，建议先用测试群组验证速率与响应表现

## 常见问题

### 机器人没有响应

请先确认消息链路本身是正常的，再检查模型请求是否成功发出。

### 返回鉴权错误

请确认 API Key 正确，并且已经注入到 LangBot 的运行环境中。

### 输出不稳定

可以先切换到更适合对话或指令跟随的模型，再根据场景调整提示词。
