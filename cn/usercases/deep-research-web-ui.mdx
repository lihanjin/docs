## 概述

Deep Research Web UI 通常需要一个可配置的大模型接口来驱动研究、总结和问答流程。

使用 AllToken 后，你可以通过统一的 API Key 和网关地址完成接入。

## 适用场景

- 你希望给研究型工作流接入统一模型服务
- 你希望把 Deep Research Web UI 的模型调用纳入 AllToken 管理
- 你希望在前端工具中快速切换不同模型

## 前置条件

- 已部署或可访问 Deep Research Web UI
- 已拥有 AllToken API Key
- 已准备好该工具所需的联网搜索服务

## 推荐配置

在 Deep Research Web UI 中，AllToken 负责 AI 模型服务部分。

如果当前版本支持 OpenAI 兼容方式，建议使用以下参数：

- API Key：`YOUR_API_KEY`
- Base URL：`https://alltoken.co/v1`
- Model：例如 `glm-4`、`deepseek-v3`、`claude-3-5-sonnet`

## 配置步骤

1. 进入 Deep Research Web UI 的设置或环境变量配置页。
2. 在 AI 模型服务一栏中，选择 OpenAI 兼容或自定义兼容接口。
3. 将模型服务地址设置为 `https://alltoken.co/v1`。
4. 填写 AllToken API Key。
5. 选择一个可用模型并保存。
6. 按该工具要求继续配置联网搜索服务，例如 Tavily、Firecrawl 或当前版本支持的其他搜索提供方。
7. 发起一次研究任务验证结果。

## 使用建议

- 建议先用一个简短问题验证模型配置，再进行长研究任务
- AI 模型服务和联网搜索服务通常是两套独立配置，二者都需要可用
- 如果你需要更稳定的长文本处理，可以优先选择上下文更大的模型

## 常见问题

### 接口能连通，但没有结果

请先检查当前模型是否具备该任务所需的能力，并确认令牌额度是否充足。

### 返回认证失败

请确认 API Key 来自 AllToken 控制台，并且仍处于启用状态。

### AI 配好了，但研究任务还是失败

这通常说明联网搜索服务尚未配置完成，或者搜索服务自身不可用。请分别检查模型服务和搜索服务两部分设置。
