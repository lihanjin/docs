## 概述

Refly 适合承载工作流、知识处理、助手编排和团队协作等 AI 场景。

将 HubTo 接入 Refly 后，你可以统一管理模型访问入口，并把 API Key、模型权限和路由策略集中到同一套网关中。

## 适用场景

- 你希望在 Refly 的工作流中调用统一模型网关
- 你希望在知识问答、流程编排或助手场景中复用多种模型
- 你希望通过单独 API Key 管理团队成员或工作区权限

## 前置条件

- 已部署或可访问 Refly
- 已拥有可用的 HubTo API Key
- 当前版本支持模型供应方配置、HTTP 请求节点或自定义服务接入

## 接入步骤

### 步骤 1：找到模型或工作流配置入口

在 Refly 中查找模型设置、Provider 配置、工作流节点配置或扩展能力入口。

### 步骤 2：填写 HubTo 连接信息

如果 Refly 支持直接配置兼容模型服务，推荐使用：

- Provider name：`HubTo`
- Base URL：`https://api.hubto.ai/v1`
- API Key：`YOUR_API_KEY`
- Model：例如 `deepseek-v3.1`、`glm-5.1`

### 步骤 3：如果需要，使用 HTTP 节点接入

如果你当前是通过工作流节点调用外部服务，也可以直接请求 HubTo 接口，把返回结果接回 Refly 的后续节点。

### 步骤 4：完成测试

建议先创建一个最小流程，例如“输入一句话并生成摘要”，确认链路可用后再接入复杂场景。

## 使用建议

- 为 Refly 单独创建一个 API Key，方便区分工作流用量
- 若工作流里有多个助手或节点，建议统一封装一个 HubTo 连接模板
- 如果需要切换不同模型，建议在 HubTo 中统一管理权限与路由

## 常见问题

### 找不到兼容模型入口

可以先检查是否支持工作流里的 HTTP 节点或自定义服务调用，这通常也能完成接入。

### 响应内容没有进入后续节点

请确认 Refly 中读取的是模型生成结果字段，而不是整个原始响应对象。

### 返回 401 或 403

请确认 API Key 正确，并且当前令牌具备目标模型权限。
