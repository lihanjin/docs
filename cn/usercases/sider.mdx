## 概述

Sider 支持接入外部模型服务。

配置 HubTo 后，你可以在 Sider 中通过统一网关访问已开通的模型。

## 适用场景

- 你希望在浏览器插件或桌面侧统一使用 HubTo
- 你希望在 Sider 中接入自定义模型服务
- 你希望把个人设备的模型调用和其他业务分开管理

## 前置条件

- 已安装并可正常使用 Sider
- 已拥有 HubTo API Key
- 已确认要使用的模型名称

## 配置步骤

### 步骤 1：打开设置

进入 Sider 的模型或 API 设置页面。

### 步骤 2：选择自定义兼容接口

优先选择 OpenAI 兼容、自定义模型服务或类似入口。

### 步骤 3：填写参数

- API Key：`YOUR_API_KEY`
- Base URL：`https://api.hubto.ai/v1`
- Model：例如 `glm-4`、`deepseek-v3`、`claude-3-5-sonnet`

### 步骤 4：验证连接

保存后发起一次测试提问，确认能够正常得到回复。

## 使用建议

- 如果 Sider 支持多个提供方，建议单独保存一套 `HubTo` 配置
- 先验证一个基础聊天模型，再切换到更长上下文或更高成本的模型
- 如果需要多人协作，建议为不同成员分配不同的 API Key

## 常见问题

### 配置保存后无法请求

请确认网络可访问 HubTo 域名，并检查 Base URL 末尾是否为 `/v1`。

### 调用成功但回复质量不符合预期

请尝试切换到更适合当前任务的模型，例如把通用聊天模型替换为更强的推理模型。
