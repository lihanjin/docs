## 概述

Chatbox 支持配置自定义模型提供方。

通过接入 Hubto，你可以在 Chatbox 中使用统一的 API Key、模型访问权限和网关地址。

## 适用场景

- 你希望在桌面端快速接入 Hubto
- 你希望在 Chatbox 中切换不同模型，但仍然统一走一个网关
- 你希望用单独的令牌管理个人设备或团队共享设备

## 前置条件

- 已安装 Chatbox
- 已拥有 Hubto API Key
- 已知需要使用的模型名称

## 配置步骤

### 步骤 1：进入模型设置

打开 Chatbox 设置页，进入模型提供方或 API 配置区域。

### 步骤 2：选择兼容类型

优先选择支持 OpenAI 兼容协议的接入方式。

### 步骤 3：填写配置信息

- API Key：`YOUR_API_KEY`
- API Host / Base URL：`https://hubto.ai/v1`
- Model：例如 `glm-4`、`deepseek-v3`、`claude-3-5-sonnet`

### 步骤 4：开始使用

保存后返回对话页，发送一条测试消息验证配置是否生效。

## 推荐验证方式

建议先从一个基础对话模型开始测试，例如发送“你好，请介绍一下你的能力”。确认返回正常后，再切换到你日常使用的模型。

## 使用建议

- 将 Chatbox 单独绑定一个 API Key，便于区分桌面端用量
- 如果你经常切换模型，可以在 Chatbox 中保存多个配置项
- 若模型列表不自动展示，可以直接手动填写模型名称

## 常见问题

### 返回 401 或鉴权错误

请确认 API Key 是否填写正确，以及前后是否包含多余空格。

### 无法列出模型

部分客户端不会自动拉取模型列表，这时可以直接手动填写模型名称。

### 返回 404 或请求地址不正确

请确认填写的是 `https://hubto.ai/v1`，而不是站点首页地址。
