## 通过三步开始使用

先在本地运行文档站点，再完成第一次自定义。

### 第一步：准备本地环境

<AccordionGroup>
  <Accordion icon="copy" title="把文档仓库克隆到本地">
    在接入流程中，如果你还没有仓库，系统会为你的文档内容创建一个 GitHub 仓库。你可以在 [dashboard](https://dashboard.mintlify.com) 中找到仓库链接。
    
    按照 GitHub 文档中的 [Cloning a repository](https://docs.github.com/en/repositories/creating-and-managing-repositories/cloning-a-repository) 指引，把仓库克隆到本地后即可开始修改和预览。
  </Accordion>
  <Accordion icon="rectangle-terminal" title="启动预览服务">
    1. 安装 Mintlify CLI：`npm i -g mint`
    2. 进入文档目录并执行：`mint dev`
    3. 打开 `http://localhost:3000` 查看实时文档预览。
    
    <Tip>你编辑文件后，预览会自动更新。</Tip>
  </Accordion>
</AccordionGroup>

### 第二步：发布你的修改

<AccordionGroup>
  <Accordion icon="github" title="安装 GitHub App">
    在 [dashboard](https://dashboard.mintlify.com/settings/organization/github-app) 安装 Mintlify GitHub App。
    
    安装后，文档变更会自动部署到你的站点，无需手动管理发布流程。
</Accordion>
<Accordion icon="palette" title="更新站点名称和颜色">
    作为第一次修改，你可以先调整站点名称和品牌色。

    1. 在编辑器中打开 `docs.json`。
    2. 把 `"name"` 字段改成你的项目名。
    3. 更新 `"colors"` 以匹配你的品牌。
    4. 保存后，前往 `http://localhost:3000` 立即查看效果。

    <Tip>可以先试着修改主色，效果会最直观。</Tip>
  </Accordion>
</AccordionGroup>

### 第三步：正式上线

<Accordion icon="rocket" title="发布文档">
  1. 提交并推送你的更改。
  2. 稍等片刻，文档站点就会更新上线。
</Accordion>
