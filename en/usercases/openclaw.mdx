## Overview

OpenClaw supports custom providers through its onboarding wizard and config files.

With AllTokens, you can point OpenClaw to a unified OpenAI-compatible gateway and manage models and API keys centrally.

## Quick setup

Set your API key:

```bash
export ALLTOKEN_API_KEY="YOUR_API_KEY"
```

Run the onboarding wizard in custom-provider mode:

```bash
openclaw onboard \
  --auth-choice custom-api-key \
  --custom-provider-id alltoken \
  --custom-compatibility openai \
  --custom-base-url "https://alltoken.aatest.online/v1" \
  --custom-model-id "glm-4" \
  --custom-api-key "$ALLTOKEN_API_KEY"
```

## Notes

- Use `https://alltoken.aatest.online/v1` as the base URL.
- Replace `glm-4` with any model that is enabled in your AllTokens account.
- If you prefer manual config, you can add a custom provider under `models.providers` and set the default model to `alltoken/glm-4`.
