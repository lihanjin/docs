description: "Connect HubTo to Maidu Radio for AI-generated audio shows."
---

## Get an API key

Create an HubTo API key first and make sure the target model is enabled.

If you want to separate this workflow from daily usage, use a dedicated key.

## Install Maidu Radio

Maidu Radio supports iOS and Android.

Install the app first, then open its settings page.

## Configure HubTo API

In the app settings, fill in:

- API Key: your HubTo key
- Base URL: `https://api.hubto.ai/v1`
- Model: a model that is enabled in your HubTo account, such as `glm-5.1` or `deepseek-v3.1`

Save the configuration and run a quick test before generating audio.

## Set voices

Open the voice settings page and choose voices for the host and assistant.

It is usually better to start with a default voice and then adjust speed, pitch, and volume.

## Create a station

Create a new station from the add button.

You can paste an RSS feed or import an OPML file.

If you already use RSSHub or WeWe RSS, you can subscribe to your favorite media sources as well.

## Generate and listen

Open the news list, pick the items you want, and generate an episode.

You can choose whether to make the content detailed, summarized, or skipped.

When the episode is ready, return to the station page and play it.

## Tips

- Validate the connection with a short item first
- Test the radio workflow and voice workflow separately
- Give Maidu Radio its own API key if you want clean usage tracking
- Prefer a stable general-purpose model if you switch models often

## Summary

Maidu Radio lets you combine subscription feeds, AI generation, and voice playback into one workflow.

With HubTo, you keep model access behind one gateway and focus on content creation.
