## 产品介绍

Hubto 是一个面向开发者的 API Key 模型网关服务。

你可以把它理解为一层统一的模型访问入口。你使用一个基础地址和 API Key，就能通过统一网关访问多种模型接口。

对于开发者，Hubto 的目标很明确：提供稳定、兼容的 API 网关接入。

## 产品功能

### 统一模型网关

Hubto 提供统一的 `/v1` 入口，并支持多类常见模型接口：

- OpenAI 兼容接口，如 `/v1/chat/completions`、`/v1/models`、`/v1/embeddings`
- OpenAI Responses 接口，如 `/v1/responses`
- Claude Messages 接口，如 `/v1/messages`
- Gemini 相关接口，如 `/v1beta/models/*`
- 音频、图片、Rerank、Realtime 等扩展能力

这意味着你可以在尽量少改客户端代码的前提下，切换模型和供应商。

### API Key 网关控制

Hubto 提供基于 API Key 的网关访问控制：

- 所有请求统一使用 API Key 鉴权
- 保持常见 SDK 与请求格式的兼容性
- 以网关方式统一接入不同模型能力

## 产品特性

### 为什么使用 Hubto

<AccordionGroup>
  <Accordion title="降低多模型接入复杂度" icon="shuffle">
    当你同时使用 OpenAI、Claude、Gemini 或其他上游时，接口格式、鉴权方式和可用模型往往并不一致。Hubto 将这些差异收敛到统一入口，减少多套 SDK 与多套配置带来的维护成本。
  </Accordion>
  <Accordion title="保持客户端改动最小" icon="wand">
    在已有请求结构基础上，你通常只需要替换基础地址和 API Key，就可以接入网关。
  </Accordion>
  <Accordion title="专注 API Key 网关控制" icon="key-round">
    产品能力聚焦在 API Key 网关层，不引入额外的平台管理能力，接入路径更清晰。
  </Accordion>
</AccordionGroup>

## 适用场景

- 为内部工具、AI 应用或代理服务提供统一模型出口
- 用 API Key 快速验证不同模型的可用性
- 在现有 OpenAI/Claude/Gemini 客户端中快速迁移到统一网关
- 作为应用的统一模型接入层
